# 📦 部署指南

## 快速部署到 GitHub Pages

### 步骤 1：推送代码到 GitHub

如果你还没有推送代码，执行以下命令：

```bash
# 添加所有文件
git add .

# 提交更改
git commit -m "部署密码管理器"

# 推送到 GitHub
git push origin main
```

### 步骤 2：启用 GitHub Pages

1. 打开你的 GitHub 仓库页面：
   ```
   https://github.com/FanYilang/password_manager
   ```

2. 点击顶部的 **Settings**（设置）标签

3. 在左侧菜单中找到 **Pages**

4. 在 **Source** 部分：
   - Branch: 选择 `main`
   - Folder: 选择 `/ (root)`
   - 点击 **Save** 按钮

5. 等待 1-2 分钟，页面会显示：
   ```
   Your site is published at https://fanyilang.github.io/password_manager/
   ```

### 步骤 3：访问你的网站

在浏览器中打开：
```
https://fanyilang.github.io/password_manager/
```

## 📱 在手机上使用

### iPhone/iPad (Safari)

1. 在 Safari 中打开网站
2. 点击底部的"分享"按钮（方框带箭头）
3. 向下滚动，选择"添加到主屏幕"
4. 输入名称（如"密码管理器"）
5. 点击"添加"

现在你可以像使用 App 一样从主屏幕打开！

### Android (Chrome)

1. 在 Chrome 中打开网站
2. 点击右上角的三个点菜单
3. 选择"添加到主屏幕"或"安装应用"
4. 输入名称
5. 点击"添加"

## 🔄 更新网站

当你修改代码后，只需：

```bash
git add .
git commit -m "更新说明"
git push origin main
```

GitHub Pages 会自动重新部署（通常需要 1-2 分钟）。

## 🔐 跨设备同步数据

由于数据存储在浏览器本地，你需要手动同步：

### 从电脑同步到手机

1. **在电脑上**：
   - 打开密码管理器
   - 点击"导出"按钮
   - 输入主密码
   - 下载备份文件（如 `password-backup-2024-01-01.json`）

2. **传输文件到手机**：
   - 通过邮件发送给自己
   - 或使用云盘（如百度网盘、OneDrive）
   - 或使用即时通讯工具（如微信"文件传输助手"）

3. **在手机上**：
   - 打开密码管理器
   - 点击"导入"按钮
   - 选择备份文件
   - 输入主密码
   - 选择合并策略（建议选"保留较新的"）

### 从手机同步到电脑

步骤相同，只是方向相反。

## ⚠️ 重要提示

1. **主密码**：
   - 请务必记住主密码
   - 主密码丢失后无法恢复数据
   - 建议使用强密码（至少 12 位，包含大小写字母、数字和符号）

2. **备份**：
   - 建议每周导出一次备份
   - 将备份文件保存在安全的地方（如加密的云盘）
   - 不要将备份文件和主密码存放在同一位置

3. **浏览器数据**：
   - 清除浏览器数据会删除所有凭证
   - 在清除数据前务必先导出备份

4. **安全性**：
   - 不要在公共电脑上使用
   - 使用完毕后记得点击"锁定"按钮
   - 不要截图包含密码的界面

## 🆘 常见问题

### Q: 为什么手机和电脑的数据不同步？

A: 数据存储在浏览器本地，需要通过"导出/导入"功能手动同步。

### Q: 忘记主密码怎么办？

A: 很遗憾，主密码无法找回。这是为了保证安全性。你需要重新设置主密码，但会丢失所有数据。

### Q: 可以在多个浏览器使用吗？

A: 可以，但每个浏览器的数据是独立的，需要手动同步。

### Q: 数据安全吗？

A: 非常安全。所有数据都在本地加密，使用军事级别的 AES-256 加密算法。

### Q: 网站打不开怎么办？

A: 检查：
1. GitHub Pages 是否已启用
2. 仓库是否设置为公开（Public）
3. 等待几分钟让部署完成
4. 清除浏览器缓存后重试
